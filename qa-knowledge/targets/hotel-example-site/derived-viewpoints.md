## DV-20260422-01: ブラウザ履歴・リロードを挟んだ条件付きUIの復元

- Parent viewpoint: 13 切替えとデータ保持
- Born from: H-20260422-01
- Rationale: 基本観点「切替えとデータ保持」はページ内の切替やシンプルなバックを想定しているが、条件付き必須UI(選択値に応じて表示・有効化が変わる入力欄)は、ブラウザバック・リロード・タブ復元を経由した際に選択値と入力欄の整合性が崩れやすい。単独のイベント遷移テストでは検出できない領域。
- Scope: フォームに条件付き必須項目がある画面全般
- Test patterns:
  - 選択値ごと (A/B/C) に入力 → 次画面 → ブラウザバック → 値と有効化状態の一致を確認
  - 有効入力 → リロード → 復元挙動を確認
  - 有効入力 → 外部リンクで離脱 → 戻る → 状態を確認
  - 確認画面で長時間放置 → バック → 状態を確認 (セッション切れ挙動と混ざりやすい)
- Promotion candidate: Yes  <!-- 他の対象でも有用そうなため基本観点への昇格候補 -->

## DV-20260422-02: エラー時のアクセシビリティ整合

- Parent viewpoint: 02 表示(UI) - エラー表示(正常系)
- Born from: H-20260422-01
- Rationale: 基本観点「エラー表示」は視認性(メッセージが出るか、どこに出るか)が中心だが、キーボード操作・aria属性・スクリーンリーダーまで含めないと、実利用でアクセシビリティ上の不具合を取りこぼす。特にフォーカス移動の問題は視覚的にテストしても気づきにくい。
- Scope: エラー表示を伴う全フォーム
- Test patterns:
  - 送信後、先頭不正項目へのフォーカス移動
  - エラーフィールドへの `aria-invalid="true"` 付与
  - `aria-describedby` によるエラーメッセージとの紐付け
  - Tab キー順序とエラー位置の整合
  - スクリーンリーダーでの読み上げ確認
- Promotion candidate: Yes
